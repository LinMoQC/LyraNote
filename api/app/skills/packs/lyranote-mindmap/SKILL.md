# LyraNote 思维导图规范

使用 `generate_mind_map` 工具生成思维导图时，遵循以下规范。

## 何时生成思维导图

- 用户说"画个思维导图"、"做个脑图"、"梳理一下结构"
- 对话中涉及复杂的概念关系，需要可视化呈现
- 总结某个主题的知识体系

## 结构规范

### 层级控制
- **最多 3 层深度**：中心主题 → 一级分支 → 二级分支
- 一级分支控制在 **4-7 个**，二级分支每组不超过 **5 个**
- 超出范围时做归类合并，避免导图过于庞杂

### 节点文字
- 一级分支：**名词短语**，2-6 个字（如"核心架构"、"应用场景"）
- 二级分支：**简短要点**，不超过 15 个字
- 避免完整句子，思维导图强调关键词

### 逻辑关系
- 同级分支之间保持**并列关系**
- 按照逻辑顺序排列：时间顺序 / 重要性顺序 / 因果顺序
- 如果有流程性质，使用序号标注

## 主题适配

| 主题类型 | 推荐结构 |
|---------|---------|
| 概念解析 | 定义 → 特征 → 分类 → 应用 |
| 技术方案 | 目标 → 架构 → 实现 → 优缺点 |
| 对比分析 | 维度 A / 维度 B / 维度 C 并列 |
| 项目规划 | 阶段 1 → 阶段 2 → 阶段 3 |
