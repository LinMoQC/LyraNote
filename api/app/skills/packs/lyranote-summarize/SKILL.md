# LyraNote 摘要生成规范

使用 `summarize_sources` 工具生成摘要时，遵循以下规范。

## 何时触发摘要

- 用户说"帮我总结一下"、"概括一下"、"给个摘要"
- 笔记本新增了多个来源，用户想了解全貌
- 深度研究结束后，对成果做概括

## 摘要结构

根据来源数量和内容复杂度，选择合适的格式：

### 单来源摘要

```markdown
## 摘要

[2-3 句核心内容概括]

## 关键要点
- 要点 1
- 要点 2
- 要点 3
```

### 多来源综合摘要

```markdown
## 综合摘要

[3-5 句整体概括，点明各来源之间的关系]

## 各来源要点

### 来源 1：[标题]
- 核心观点 A
- 核心观点 B

### 来源 2：[标题]
- 核心观点 C

## 共同发现与差异
- 一致之处：...
- 不同视角：...
```

## 质量标准

1. **信息密度高**：避免空洞措辞，每句话都传递实质信息
2. **保持客观**：忠实于原文，不添加主观判断
3. **保留关键数据**：原文中的数字、比例、日期等关键数据必须保留
4. **标注来源**：每个关键观点标注来源编号 [来源N]
5. **长度适中**：摘要篇幅控制在原文 20%-30%
