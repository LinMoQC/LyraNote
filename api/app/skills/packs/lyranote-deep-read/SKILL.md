# LyraNote 深度阅读规范

使用 `deep_read_sources` 工具对来源进行深度阅读分析时，遵循以下规范。

## 何时触发深度阅读

- 用户说"帮我深度分析一下这篇文档"、"仔细读一下"
- 用户对某个来源有具体问题，需要深入内容回答
- 来源内容较长，需要分段提炼核心论点

## 分析输出结构

```markdown
## 文档概述
[1-2 句话概括文档的主题、作者意图和核心结论]

## 核心论点分析
### 论点 1：[论点名称]
- **原文依据**：引用关键段落
- **分析解读**：该论点的意义和影响

### 论点 2：[论点名称]
- **原文依据**：引用关键段落
- **分析解读**：该论点的意义和影响

## 方法论与数据
- 使用了哪些方法 / 数据来源
- 数据是否充分支撑结论

## 局限性与启发
- 文档的局限或未覆盖之处
- 对后续研究的启发
```

## 质量标准

1. **区分事实与观点**：明确标注哪些是文档原文的事实，哪些是分析性评论
2. **引用原文**：关键论断必须引用原文片段作为依据
3. **批判性思维**：指出论证中的薄弱环节或逻辑漏洞
4. **关联上下文**：将文档内容与用户知识库中的其他资料做关联
