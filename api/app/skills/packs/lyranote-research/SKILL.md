# LyraNote 研究工作流

当用户提出研究性问题时，遵循以下工作流：

## 标准研究流程

1. **优先检索本地知识库**（`search_notebook_knowledge`）
   - 知识库中已有资料时，先从这里获取答案
   - 检索结果作为分析的基础素材

2. **联网补充最新信息**（`web_search`）
   - 用户明确要求联网时，或话题涉及近期动态时必须联网
   - 将网络结果与本地资料交叉对比，识别差异

3. **综合分析与引用**
   - 合并多来源信息，给出有深度的分析
   - 对每个关键观点标注来源编号 [来源N]
   - 指出来源之间的一致性或矛盾点

## 输出质量标准

- 回答要有结构：背景 → 核心发现 → 分析 → 结论
- 长回答使用 Markdown 标题和列表，短回答保持自然段落
- 不确定的内容明确标注"暂不确定"或"需进一步验证"
- 避免重复已在工具输出中展示的内容

## 何时建议用户创建笔记

- 研究产生了新的重要结论时，主动提议：「要把这个分析保存为笔记吗？」
- 用户反复问到同一主题，说明该主题值得系统化整理
