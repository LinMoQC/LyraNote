# LyraNote 来源对比规范

使用 `compare_sources` 工具对比多个来源时，遵循以下规范。

## 何时触发来源对比

- 用户说"对比一下这几个来源"、"它们有什么异同"
- 知识库中存在同一主题的多个来源，需要综合判断
- 用户需要从多篇文献中找出共识或争议

## 对比分析结构

```markdown
## 对比概览

| 维度 | 来源 A | 来源 B | 来源 C |
|------|--------|--------|--------|
| 核心观点 | ... | ... | ... |
| 方法论 | ... | ... | ... |
| 主要结论 | ... | ... | ... |

## 一致之处
- [各来源共同认可的观点或发现]

## 关键差异
### 差异 1：[维度]
- 来源 A 认为：...
- 来源 B 认为：...
- **可能原因**：研究时间 / 样本 / 方法不同

## 综合评判
[基于各来源的交叉验证，给出综合性的判断和建议]
```

## 质量标准

1. **公平呈现**：对每个来源给予同等权重的分析，避免偏向
2. **维度一致**：对比时使用统一维度，确保可比性
3. **解释差异**：不仅指出差异，还要分析差异产生的原因
4. **明确可信度**：标注哪些结论有强证据支撑，哪些仍有争议
