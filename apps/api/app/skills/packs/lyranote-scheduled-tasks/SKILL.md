# LyraNote 定时任务规范

使用 `create_scheduled_task` 工具创建定时任务时，遵循以下规范。

## 创建前必须确认（最重要）

**禁止在信息不完整时直接调用工具。** 以下四项为必填项，用户未明确提供时必须先询问：

1. **任务名称**（name）：用户没说名字时，提议一个并确认
2. **关注主题**（topic）：用户没说关注什么内容时，必须询问
3. **执行频率**（schedule）：用户没说多久执行一次时，询问或建议默认值
4. **投递方式**（delivery）：用户没说结果怎么接收时，询问或建议默认值

**典型场景：** 用户说"帮我订阅 https://xxx.com/rss.xml" —— 此时只有订阅源，缺少任务名称、关注主题、频率等，应回复：
> 「好的，我来帮你创建定时任务。请问：
> 1. 这个任务叫什么名字？（例如：XX 资讯日报）
> 2. 你主要关注哪些主题方向？
> 3. 多久执行一次？（每天 / 每 3 天 / 每周）
> 4. 结果写入笔记还是发送邮件？」

## 何时创建定时任务

- 用户说"每天帮我关注..."、"定期监控..."、"订阅..."
- 用户需要周期性的信息采集、资讯整理
- 用户提供了 RSS 订阅源，想自动跟踪

## 参数选择指南

### 任务名称
- 简洁具体，体现内容和频率
- 好：「AI 前沿日报」「安全漏洞周刊」
- 差：「每天的任务」「定期报告」

### 主题关键词
- 包含 2-4 个核心关键词，用空格分隔
- 避免过于宽泛（"科技"）或过于狭窄（"GPT-5 API 价格变动"）
- 好：「AI 大模型 最新进展 技术突破」

### 订阅源（feed_urls）
- 当用户提供了 RSS/Atom 链接时，传入 `feed_urls` 参数
- 订阅源和主题可同时使用：优先从订阅源获取，不足时用主题搜索补充
- 未提供订阅源时，系统通过 Perplexity/Tavily 搜索

### 执行频率
- 新闻资讯类：`daily`（每天）
- 行业报告类：`weekly`（每周）或 `every_3_days`
- 趋势观察类：`biweekly`（每两周）或 `monthly`

### 投递方式
- 默认使用 `note`（保存为笔记）
- 用户明确要求邮件时使用 `email` 或 `both`
- 使用邮件投递时需确认用户已配置邮箱

## 创建后的提示

创建成功后，告知用户：
1. 任务将按设定频率自动执行
2. 可以在「定时任务」页面管理（启用/暂停/删除）
3. 首次执行时间
