# LyraNote 网络搜索策略

`web_search` 工具现已支持多 provider、全文提取和自动重排序。请遵循以下五步框架使用它。

## Step 1 — 查询规划

- **复杂问题先拆分**：将宽泛问题拆解为 2-3 个聚焦子查询，分别调用 `web_search`。
  - 例："大宗商品涨价对通胀的影响" → 先搜"大宗商品价格走势 2024"，再搜"通胀与大宗商品相关性研究"
- **每次 query 聚焦单一角度**，避免过于宽泛（如"人工智能所有内容"）
- **技术内容优先用英文 query**：英文资料覆盖度更广、时效性更强
- **先查本地知识库**（`search_notebook_knowledge`），再联网补充——避免重复搜索已有内容

## Step 2 — 搜索参数选择

根据任务类型选择合适的参数：

| 场景 | 推荐参数 |
|------|---------|
| 最新新闻 / 动态 | `days=7` 或 `days=1` |
| 近期研究 / 报告 | `days=30` |
| 深度技术文档 | `search_depth="advanced"` |
| 完整论文 / 长文 | `fetch_full_content=true` |
| 快速概念查询 | 默认参数即可 |

`fetch_full_content=true` 会用 Jina Reader 提取排名前 2 的 URL 全文，耗时更长但内容更丰富，仅在确实需要完整正文时使用。

## Step 3 — 内容质量判断

工具输出末尾会有 **[搜索质量评估]** 字段，包含结果数量和最高相关度：

- **相关度 < 40%**：说明当前 query 偏离主题，换用不同关键词角度后重试
- **结果数 < 3**：信息覆盖不足，考虑换 query 或换语言（中/英互换）
- **内容过于简短**（只有标题，无正文）：设置 `fetch_full_content=true` 提取全文

## Step 4 — 循环修正规则

- **同一 query 不重复调用**——如需再搜，必须修改关键词或调整参数
- **最多搜索 3 轮**后，即使信息仍不完整也应基于已有内容作答，并告知用户信息局限性
- 如果多轮搜索均无高质量结果，坦诚说明"当前可查信息有限"，不要凭空推测

## Step 5 — 结果整合

- 合并多次搜索结果时，优先使用相关度最高的内容
- 对矛盾信息标注各来源观点，不自行判断哪个"更正确"
- Perplexity "综合分析" 或 Tavily "AI Answer" 字段是高质量综合内容，可直接作为主要依据
- 每个关键结论引用对应来源 [来源N]
